MODULAR MONOLITH ARCHITECTURE (MICROSERVICE-READY)

OVERVIEW:
The system follows a Modular Monolith architecture using the MERN stack.
It remains a single deployable application but is internally structured as independent domain modules.

Key Principles:

* Domain-driven modularization
* Separation of concerns
* Event-driven internal communication
* Microservice-ready boundaries

6. INTERNAL EVENT-DRIVEN DESIGN

---

events/
|
|-- eventBus.js                 # Event dispatcher
|-- events.js                   # Event definitions

EXAMPLE FLOW:

Expense Created
-> Emit EVENT: EXPENSE_CREATED
-> Update balances (groups module)
-> Update analytics
-> Trigger notifications (Socket.io)

7. DATABASE DESIGN

---

Collections:

* users
* groups
* expenses
* funds
* settlements
* notifications

Rules:

* Each module owns its schema
* No direct cross-module DB access
* Interactions must go through services
